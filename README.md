# Análise de Dados

## Sobre o Projeto

Este projeto simula o banco de dados de uma distribuidora de peças para motocicletas, com foco na análise de vendas, comportamento de clientes e desempenho de produtos.

O objetivo é aplicar conceitos de modelagem de dados, SQL e análise para gerar insights que auxiliem na tomada de decisão.

---

## Tecnologias Utilizadas

* SQL (MySQL)
* Modelagem de Dados Relacional
* Looker Studio (para visualização - em desenvolvimento)
* Git e GitHub

---

## Estrutura do Banco de Dados

O banco foi modelado utilizando conceitos de normalização e integridade referencial.

### Tabelas:

* **clientes** → informações dos clientes (oficinas e pessoas físicas)
* **produtos** → catálogo de peças de motocicletas
* **categorias** → classificação dos produtos
* **pedidos** → registros de compras realizadas
* **itens_pedido** → itens contidos em cada pedido

---

## Relacionamentos

* Um cliente pode realizar vários pedidos
* Um pedido pode conter vários produtos
* Um produto pertence a uma categoria

---

## Análises Realizadas

* Faturamento total
* Faturamento por produto
* Faturamento por mês
* 5 Produtos mais vendidos
* Produtos mais lucrativos
